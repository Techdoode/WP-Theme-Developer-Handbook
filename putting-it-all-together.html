Basic Files

At an absolute minimum, a valid WordPress theme must have two files: style.css and index.php. However, in the interest of flexibility and modular design, you'll want to go beyond the minimum. We'll build our simple theme with the following set of files:
  • style.css
  • index.php
  • header.php
  • sidebar.php
  • footer.php

What is the Index Template?

The Index Template, index.php, along with style.css, is one of two files required by a WordPress theme. It serves a similar function to index files in a standard website: a repository for the main page markup. If the theme has no other template files, WordPress will use index.php for everything. In more complex websites, this file is the final fallback to render the entire website if it can't find a particular file in the template hierarchy. While it is possible to put all your markup in index.php, breaking up the main logical sections of the page into separate files allows greater degree of flexibility and modularity. These files are then included at the appropriate places in index.php. Additionally, the Index Template contains The Loop, which is the engine that drives the listing of your Posts. However this file serves additional functions in WordPress. Firstly, it is one of two required theme files. Secondly, it serves as a pattern for several optional files. Lastly, it is the fall-back file in case the optional files are missing.
